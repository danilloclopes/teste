1. Objetivo do Projeto O objetivo deste segundo
trabalho é dar continuidade ao projeto "Construção de Sistema Web Seguro
com Arquitetura MVC", modernizando a camada de apresentação (Front-end)
através da aplicação do modelo React. Os alunos deverão refatorar de
forma inteligente a aplicação construída anteriormente, substituindo a
renderização de páginas JSP por uma arquitetura desacoplada onde o Front-end
(React) e o Back-end (Java Servlets) operam de forma independente. O
foco deve ser explorar as vantagens do React, como a modularização, o reuso de
componentes, e o gerenciamento de estados para criar uma experiência de usuário
mais dinâmica e fluida.



2. Separação de Responsabilidades (Front-end vs.
Back-end) No modelo moderno de desenvolvimento de aplicações, os objetivos
das camadas se separam claramente.


Front-end
(React): O foco será criar uma interface de usuário totalmente
funcional, acessível, com layouts responsivos e de alta performance.
Back-end
(Java Servlets): O back-end passará a atuar exclusivamente como uma
API. Em vez de utilizar o método RequestDispatcher.forward() para enviar
atributos a uma página JSP, os Servlets deverão retornar os dados no
formato JSON. O foco do back-end passa a ser fornecer uma
arquitetura confiável, segura e eficiente para alimentar o front-end.



3. Modularização e Reuso de Componentes (React) Uma
das maiores vantagens do React é a sua arquitetura baseada em componentes. No
projeto, a equipe deve refatorar as antigas páginas HTML/JSP em pequenas peças
de interface independentes e reutilizáveis (ex: cabeçalhos, rodapés, botões,
formulários, tabelas de dados). O sistema não deve possuir redundância de
código visual; se um elemento se repete em várias telas, ele obrigatoriamente
deve ser um componente React modularizado.



4. Interatividade Dinâmica: Estado e Eventos O
sistema deve tirar proveito do controle de Eventos e Estado (State) do React.
Em vez de recarregar a tela inteira a cada requisição (comportamento típico do
modelo antigo com Servlets e JSP), a interface deve atualizar apenas os
componentes que sofreram alteração (ex: edição, filtragem de dados e
renderização condicional).



5. Comunicação Assíncrona e APIs Web A integração
entre o React e os Servlets Java deve ser feita através de JavaScript
Assíncrono e Promises. O front-end deverá realizar requisições assíncronas
(AJAX / Fetch API) para a API Java, enviando e recebendo dados estruturados em JSON
para atualizar as telas dinamicamente sem interromper a navegação do usuário.



6. Ferramentas, Testes e Hospedagem Para este segundo
trabalho, a equipe deverá incorporar ferramentas indispensáveis para o mercado
moderno de desenvolvimento:


Testes
de API: A comunicação do novo Back-end (API REST/Servlets) deve ser
testada e documentada obrigatoriamente utilizando o Bruno (usebruno),
garantindo que as requisições GET, POST, PUT e DELETE estejam funcionando
perfeitamente antes de integrá-las ao React.
Hospedagem
Front-end: O código Front-end em React deverá ser publicado
gratuitamente utilizando o GitHub Pages. A equipe deve configurar o
repositório para que o deploy da aplicação seja feito de forma
automatizada diretamente pelo GitHub.
Controle
de Versão e Trabalho em Equipe: O uso do Git no mundo real será
rigorosamente avaliado. A equipe deve continuar utilizando o GitHub com branches,
commits com sentido lógico, Pull Requests pequenos e sem
quebrar a build.
